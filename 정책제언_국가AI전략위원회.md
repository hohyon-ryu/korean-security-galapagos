## 1. 법률 개정 제언

### 1-1. 전자금융거래법

**현황:** 2025년 12월 16일 개정안이 공포되어 2026년 12월 17일 시행 예정. 업종 구조를 7개에서 3개(자금이체업·대금결제업·결제대행업)로 통합하고 '지급지시전달업(마이페이먼트)'을 신설.

| 개정 필요 사항 | 담당 | 구체적 행동 | 시기 |
|---|---|---|---|
| **전자금융감독규정 제15조(보안프로그램 설치)** 삭제/완화 | 금융위원회 | 클라이언트 보안프로그램 강제 설치 조항을 서버사이드 보안(FDS, 행동 분석)으로 대체. "금융회사는 이용자 단말기에 보안프로그램 설치를 요구하지 않을 수 있다"는 예외 조항을 원칙으로 전환 | 2027년 상반기 |
| **전자금융감독규정 제34조(전자적 전송)** 개정 | 금융위원회 | 해외 결제 네트워크(Visa/Mastercard)와의 직접 연동을 허용하는 규정 신설. 외국 전자금융업자의 국내 등록 절차를 간소화 | 2027년 |
| **시행령 내 지급지시전달업 등록요건** 명확화 | 금융위원회 | 최소자본금 3억 원, 해외 사업자의 국내 지점 설치 의무 완화, API 표준 규격 제정 | 2026년 12월 시행 전 |
| **전자금융감독규정 제14조의2(망분리)** 완화 지속 | 금융위원회·금융감독원 | SaaS 활용 범위를 비중요 업무에서 중요 업무까지 단계적 확대. 2025년 1월 개정된 시행세칙을 바탕으로 2단계(금융보안 법제 개편) 추진 | 2027-2028년 |

**참고:** 전자금융감독규정 개정 3단계 로드맵 — 1단계: 규정 개선(완료), 2단계: 금융보안 법제 개편, 3단계: 자율보안 체계 전환

## 2. EU PSD2/PSD3 벤치마킹

### EU PSD3/PSR 핵심 내용 (2025년 11월 잠정 합의, 2028년 시행 예상)

| EU PSD3/PSR 요건 | 한국 현황 | 한국형 적용 방안 |
|---|---|---|
| **강한 고객 인증(SCA)** 의무화 — 2가지 이상 인증 요소 필수 | 한국은 이미 다중 인증 적용 중이나 SMS OTP 의존도 높음 | SMS OTP를 피싱 방지형 인증(FIDO2)으로 전환 의무화 |
| **오픈뱅킹 API 무상 제공 의무** — 계좌정보서비스(AISP)·지급개시서비스(PISP) | 한국 오픈뱅킹은 은행 자율 합의 기반. 법적 의무 아님 | 전자금융거래법에 '오픈API 제공 의무' 조항 신설 |
| **제3자 결제서비스 제공자(TPP)** 의 시장 접근 보장 | VAN/PG 구조가 중간 계층으로 작용하여 직접 접근 제한 | 결제대행업 등록 외국 사업자의 은행 결제망 직접 접근 허용 |
| **이용자 보호 강화** — 무권한 거래 시 즉시 환불 의무 | 전자금융거래법상 이용자 보호 규정 존재하나 실효성 미흡 | 무권한 거래 발생 시 D+1 영업일 내 선환급 의무화 |
| **대시보드(Permission Dashboard)** — 이용자가 데이터 접근 권한을 한 곳에서 관리 | 해당 제도 없음 | 마이데이터 포털에 권한 관리 대시보드 기능 추가 |

### 한국형 'K-PSD3' 프레임워크 제안

**금융위원회**가 주도하여 2027년까지 다음을 포함하는 「개방형 금융서비스 기본계획」을 수립:

1. 은행의 오픈API 제공을 **법적 의무**로 격상 (현행 자율 합의 → 법률 명시)
2. 지급지시전달업자(PISP)의 은행 결제망 **무차별 접근권** 보장
3. 계좌정보서비스(AISP) 사업자에 대한 **독립 라이선스** 신설
4. **API 표준 규격**을 금융보안원이 제정하고 정기 업데이트

### 4-2. 금융감독원 (FSS)

## 7. VAN/PG 결제 구조 개혁

### 7-1. 현재 구조의 문제

한국의 온라인 결제는 **가맹점 → PG사 → VAN사 → 카드사** 4단계 구조. 글로벌 표준은 **가맹점 → 결제서비스제공자(PSP) → 카드네트워크** 2~3단계. 한국의 다층 구조는 수수료 증가, 외국 사업자 진입 장벽, 혁신 저해의 원인.

**Stripe가 한국에서 안 되는 이유:**
- 한국 법인/사업자가 Stripe 계정 개설 불가 (한국 금융 규제 미충족)
- 카드정보를 가맹점이 보유하면 불법 → 반드시 PG사 경유
- VAN사를 거치는 중간 계층이 필수

### 7-2. 개혁 방안

| 과제 | 구체적 행동 | 담당 | 시기 |
|---|---|---|---|
| **카드네트워크 직접 연동 허용** | PG사가 VAN을 거치지 않고 Visa/Mastercard 네트워크에 직접 연결할 수 있는 규정 마련. 여신전문금융업법 시행령 개정 | 금융위원회 | 2027년 |
| **외국 PSP 등록 절차 간소화** | Stripe, Adyen 등이 한국 내 결제대행업 등록 시, 본국 라이선스·PCI DSS 인증을 인정하는 상호인정 협정 체결 | 금융위원회 | 2027-2028년 |
| **VAN사 역할 재정의** | VAN사의 단순 중계 기능을 데이터 분석·부가가치 서비스로 전환 유도. VAN 수수료 상한제 도입 | 금융위원회 | 2027년 |
| **토큰화(Tokenization) 의무화** | 카드 번호 대신 토큰을 사용하는 EMV 토큰화를 의무화하여, 가맹점의 카드정보 보유 금지 조항과 양립 가능한 글로벌 결제 환경 조성 | 금융위원회·카드사 | 2028년 |

### 7-3. 벤치마킹: 인도 UPI

인도의 UPI는 2024년 약 1,720억 건의 거래를 처리하며 세계 실시간 결제 시스템의 49%를 차지. 핵심 성공 요인:
- **개방형 아키텍처**: 핀테크가 은행 계좌에 직접 접근하는 앱 개발 가능
- **무료 인프라**: 정부가 결제 인프라를 공공재로 운영
- **독점 방지**: 개별 사업자의 시장 점유율 상한제 적용

한국도 오픈뱅킹을 법적 의무로 격상하고, 결제 인프라의 공공성을 강화하는 방향으로 나아가야 함.

## 8. 휴대폰 본인인증 독점 해소

### 8-1. 현재 독점 구조

- SKT·KT·LGU+ 3사가 2012년 12월 본인확인기관으로 지정된 이래 사실상 독점
- 인증 건당 수수료: SKT 23원, KT 30원, LGU+ 30원
- 연간 시장 규모: 최소 200~300억 원 (5년간 최소 1,000억 원)
- PASS 앱 가입자 3,700만 명 — 높은 이용자 접근성을 빌미로 독점 유지
- 2017년 이후 신용카드사(삼성·신한·현대·NH농협 등 7개사) 추가 지정되었으나, 실제 시장 점유율은 여전히 통신사 편중

### 8-2. 개혁 방안

| 과제 | 구체적 행동 | 담당 | 시기 |
|---|---|---|---|
| **본인확인기관 지정 확대** | DID 기반 인증기관, 핀테크 기업(토스, 카카오페이 등)을 본인확인기관으로 추가 지정 | 방송통신위원회·과기정통부 | 2027년 |
| **본인확인 수수료 투명화** | 건당 수수료 공시 의무화 및 상한제 도입. 현재 23~30원의 불투명한 가격 구조 개선 | 방송통신위원회 | 2027년 |
| **이메일+MFA 인증 허용** | 글로벌 표준인 이메일 기반 계정 생성 + MFA(FIDO2/TOTP)를 본인확인 수단으로 법적 인정. 현재 서비스 가입 시 휴대폰 인증 의존도 과도 | 과기정통부 | 2028년 |
| **DID 기반 자기주권 인증 도입** | 2025년 전면 시행된 모바일 신분증(DID 기반)을 온라인 본인확인 수단으로 확대. 1회용 QR 인증으로 개인정보 최소 노출 | 행정안전부·과기정통부 | 2027-2028년 |

### 8-3. 벤치마킹: 에스토니아 e-Residency / EU eIDAS

에스토니아는 국가 차원의 디지털 ID 인프라를 구축하여 통신사 의존 없이 전자 인증·서명이 가능. EU의 eIDAS 규정은 회원국 간 전자신원증명의 상호인정을 의무화. 한국도 DID 기반 모바일 신분증을 국가 공인 전자신원증명으로 격상하여 통신사 독점을 근본적으로 해소해야 함.

| **FDS 고도화** | AI 기반 실시간 이상거래탐지 투자 확대. 클라이언트 보안프로그램 의존도를 연 10%씩 축소하고 서버사이드 탐지로 전환 | 영국 Barclays: AI FDS로 사기 탐지율 40% 향상 | 2026-2028년 |
