# 왜 한국에서만 은행 업무를 하려면 이상한 프로그램을 깔아야 할까?

> 한국의 금융 보안 시스템은 보이스피싱과 금융사기에 대응하면서 생긴 것이다. 문제는, **실제 사기 수법에는 무력하면서 오히려 북한 해커에게 문을 열어줬다**는 점이다. 그리고 이 시스템을 유지하는 데 이해관계가 걸린 업체들이, 규제의 보호 아래 이권 구조를 형성하고 있다.

## 1. 처음엔 이유가 있었다

한국의 금융 보안 시스템이 이렇게 된 데는 이유가 있다. 보이스피싱과 금융사기 피해가 진짜 심각했다.

**보이스피싱 피해 규모:**
- 2023년 피해액 약 **1조 965억 원** (경찰청 통계)
- 하루 평균 **30억 원**이 사기로 빠져나간다
- 피해자 상당수가 노년층, 생활비·전 재산을 잃는 경우 빈번

정부 입장에서는 "뭐라도 해야 한다"는 압박이 엄청났다. 그래서 나온 대책이:

- 키보드 입력을 해커가 훔쳐보지 못하게 → **키보드 보안 프로그램** (TouchEn nxKey)
- 악성 프로그램이 은행 화면을 조작하지 못하게 → **방화벽 프로그램** (AhnLab Safe Transaction)
- 사용자가 진짜 본인인지 확인 → **공인인증서**, 보안카드, OTP
- 사기범의 IP를 추적하기 위해 → **IP 수집 프로그램** (IPinside)

출발점 자체는 이해할 수 있다. **문제는 이 대책들이 실제 사기 수법과 전혀 맞지 않는다는 것이다.**

## 2. 왜 엉뚱한 방향으로 갔을까?

### 보이스피싱은 이렇게 작동한다

보이스피싱의 실제 공격 단계를 보자:

```
1단계: 전화가 온다
   "검찰입니다. 당신 계좌가 범죄에 연루됐습니다."
   "은행입니다. 대출 금리를 낮춰드릴 수 있습니다."

2단계: 피해자가 속는다
   공포·욕심·신뢰 — 사회공학적 조작

3단계: 피해자가 직접 행동한다
   (a) 직접 돈을 이체하거나
   (b) 원격제어 앱(팀뷰어, AnyDesk)을 직접 설치하거나
   (c) 악성 앱을 직접 설치한다

4단계: 돈이 빠져나간다
```

핵심: **피해자가 속아서 자기 손으로 직접 하는 것이다.**

### 한국의 보안 프로그램이 막을 수 있는 것

| 보이스피싱 공격 방식 | 키보드 보안이 막나? | 방화벽이 막나? | 공인인증서가 막나? |
|---|---|---|---|
| 피해자가 **직접** 이체 버튼을 누름 | 못 막음 | 못 막음 | 못 막음 (본인이 인증하니까) |
| 피해자가 **직접** 원격제어 앱을 설치 | 못 막음 | 못 막음 | 못 막음 |
| 피해자가 **직접** 악성 앱을 설치 | 못 막음 | 못 막음 | 못 막음 |
| 해커가 몰래 키보드 입력을 훔침 | **이것만 막음** | — | — |

보이스피싱 피해의 대부분은 "피해자가 속아서 직접 행동하는" 유형이다. 키보드 보안 프로그램이 막을 수 있는 건 "해커가 사용자 몰래 키보드 입력을 가로채는" 유형뿐인데, 이건 보이스피싱의 주요 수법이 아니다.

**비유하자면:** 집에 도둑이 현관문을 두드려서 "택배입니다"라고 속이면, 집주인이 직접 문을 열어준다. 이때 창문에 아무리 강화유리를 달아봤자 소용없다. **그런데 한국 정부의 대책은 "창문에 강화유리를 의무적으로 달아라"인 것이다.**

### 실제로 효과 있는 건 따로 있다

보이스피싱에 진짜 효과 있는 대책은 전부 **서버(은행) 쪽**에서 하는 것이다:

| 대책 | 작동 방식 | 어디서 작동? |
|------|-----------|-------------|
| **이상거래탐지(FDS)** | "이 사람이 평소에 안 하던 패턴으로 큰 돈을 보낸다" → 자동 차단 | 은행 서버 |
| **지연이체** | 큰 금액 이체 시 30분~1시간 후 실행 → 냉각기간 | 은행 서버 |
| **대포통장 탐지** | 짧은 시간에 여러 계좌에서 돈이 들어오는 패턴 감지 | 은행 서버 |
| **AI 통화 분석** | 통화 중 "검찰", "계좌 동결" 등 키워드 감지 → 경고 | 통신사 서버 |

이 중 어느 것도 **사용자 PC에 프로그램을 설치할 필요가 없다.**

### 그래서 진짜 문제는 뭔가?

한국과 글로벌의 접근 방식 차이를 한 줄로 요약하면:

> **한국:** "사용자 PC가 위험하니까, PC를 통제하겠다" (프로그램 강제 설치)
> **글로벌:** "사용자 PC는 어차피 통제할 수 없으니까, 서버에서 막겠다"

한국 방식의 숨겨진 동기가 여기 있다:

```
[한국의 실제 논리]

사고가 나면 누구 책임인가?
  → 은행: "우리는 보안 프로그램 설치하라고 했잖아요"
  → 금감원: "은행이 보안 조치를 했으니 면책"
  → 결론: 사용자 책임

보안 프로그램의 진짜 역할:
  ✗ 해킹을 막는 것 (실제로 못 막음)
  ✓ 은행의 면책 근거를 만드는 것
```

다시 비유하자면: 집에 도둑이 들었다. 경찰이 와서 "강화유리 설치했습니까?" → "아니요" → "그럼 본인 책임입니다." 도둑이 현관문으로 들어왔는데도.

**글로벌(EU)의 접근은 정반대다:**

```
[EU PSD2의 논리]

사고가 나면 누구 책임인가?
  → 금융기관이 충분한 보안 조치를 했는가?
  → 서버측 이상거래탐지, 토큰화, 강력 인증을 구현했는가?
  → 아니라면 → 금융기관 책임
```

**책임을 금융기관에 두면**, 은행은 "프로그램 깔라고 했으니 우리 잘못 아니야"가 통하지 않는다. 은행이 직접 서버 보안을 강화할 수밖에 없다. 사용자 PC에 뭘 깔든 안 깔든, 은행이 책임지니까.

## 3. 한국에서만 볼 수 있는 것들

### 3.1 공인인증서 → 공동인증서 (이름만 바뀜)

**1999년:** 정부가 공인인증서를 만들었다. Internet Explorer의 ActiveX로 구현.

**2014년:** 박근혜 대통령 "천송이 코트" 발언 — "중국인들이 공인인증서 때문에 한국 쇼핑몰에서 못 산다." (실제로는 사실이 아니었지만 국민적 불만에 불을 붙임)

**2020년 12월:** 법을 바꿔서 "공인" → "공동"으로 이름이 바뀜.

**2026년 현재:** 이름만 바뀌었다. USB에 파일 저장하고, 브라우저 플러그인 깔고 — 기술 구조는 27년 전 그대로다.

간편인증서(카카오, 네이버, PASS)가 나오면서 **스마트폰에서는** 많이 편해졌다. 하지만 **PC에서 은행 업무를 하려면?** 여전히 지옥이다.

### 3.2 신용카드 비밀번호 앞 2자리

한국에서 온라인 결제할 때 카드 비밀번호 4자리 중 **앞 2자리만** 입력한다. 세계 어디에도 이런 방식은 없다.

왜 2자리만? 명확한 보안적 근거가 없다. "4자리 다 치면 유출 위험이 크니까 2자리만"이라는 논리인데, 2자리만 유출돼도 조합 가능한 경우의 수가 100개밖에 안 된다.

### 3.3 가상 키패드 — 숫자 순서를 매번 섞어놓기

비밀번호 칠 때 화면에 키패드가 뜨는데, 1~9 숫자가 **매번 다른 순서로 배열**된다. 키로거(키보드 입력 훔쳐보는 악성코드)를 막겠다는 건데:

**아이러니:** 이 가상 키패드를 제공하는 프로그램(TouchEn nxKey) 자체에 키로깅 취약점이 발견됐다. 도둑을 막으려고 고용한 경비원이 도둑이었던 셈.

### 3.4 보안카드

지갑에 넣고 다니는 작은 카드에 30개의 4자리 숫자가 인쇄되어 있다. 은행 거래할 때 "3번, 15번 코드를 입력하세요" 하는 식.

- 카드 잃어버리면? → 거래 불가
- 사진 찍어서 폰에 저장하면? → 폰 해킹되면 끝
- 전화 사기범이 "보안카드 번호 불러주세요"라고 하면? → 불러주는 사람이 실제로 있다

### 3.5 SMS 인증번호 만능주의

모든 것에 문자 인증번호. 계좌이체, 카드 등록, 비밀번호 변경, 인증서 갱신 — 전부.

문제: 미국 국립표준기술연구소(NIST)는 **2016년부터** SMS 인증을 권장하지 않는다. 이유:
- **SIM 스와핑:** 사기범이 통신사에 전화해서 "제 유심을 새 폰으로 옮겨주세요"라고 하면, 인증 문자가 사기범 폰으로 간다
- **SS7 취약점:** 통신 프로토콜 자체의 결함으로 문자 메시지를 가로챌 수 있다

### 3.6 30만원 한도 (폐지됐지만)

2014년까지 **30만 원 이상** 온라인 결제하려면 공인인증서가 필수였다. 법은 바뀌었지만 은행들이 자체 규정으로 수년간 계속 요구했다.

## 4. 그 프로그램들의 정체

한국 은행 웹사이트에 접속하면 **한꺼번에 5개 이상**의 프로그램을 설치하라고 뜬다. KAIST 연구에 따르면 한국인 PC에는 평균 **9개**의 이런 프로그램이 깔려 있다.

이 프로그램들이 실제로 뭘 하는지 하나씩 보자.

### TouchEn nxKey (라온시큐어) — "키보드 보안"

**은행이 말하는 용도:** 해커가 키보드 입력을 훔쳐보지 못하게 막는 프로그램

**실제로 하는 일:** 이 프로그램은 키로거를 막기 위해 **자기 자신이 키로거가 된다**.

쉽게 설명하면:
1. 당신이 비밀번호를 친다
2. 이 프로그램이 **먼저** 키보드 입력을 가로챈다 (운영체제 깊숙한 곳에서)
3. 암호화해서 은행 웹사이트에 전달한다
4. "나(TouchEn)만 키보드 입력을 가로챌 수 있게 했으니까 해커는 못 가로챈다"는 논리

**문제:** 2023년 보안 연구자 Wladimir Palant가 7개의 심각한 취약점을 발견했다. **악성 웹사이트가 이 프로그램을 이용해서 키보드 입력을 전부 훔칠 수 있었다.** 키로거를 막으려고 깐 프로그램이 오히려 해커에게 키로깅 통로를 제공한 것이다.

Chrome 웹스토어에서 이 프로그램의 평점은 **1.3점**(5점 만점). 리뷰는 "제발 없애달라"로 가득하다.

### IPinside (인터리젠) — "IP 확인"

**은행이 말하는 용도:** 접속자의 실제 IP 주소를 확인해서 사기를 방지

**실제로 하는 일:** IP 주소 확인은 핑계고, 실제로는 당신의 PC를 **속속들이 조사**한다:
- 하드 드라이브 시리얼 번호
- 설치된 프로그램 목록
- 네트워크 설정 전체
- 가상머신(VirtualBox 등) 사용 여부
- 기타 시스템 정보

보안 연구자 Palant는 이 프로그램을 이렇게 평가했다: **"이것은 의무 설치 스파이웨어(mandatory spyware)다."**

더 문제인 것: 이 프로그램을 만든 인터리젠은 **빅데이터 수집·분석 전문 회사**다. 은행·보험사·정부기관에 데이터를 판매하는 것이 본업이다.

### Veraport (위즈베라) — "보안 프로그램 관리"

**은행이 말하는 용도:** 필요한 보안 프로그램들을 자동으로 설치·관리해주는 프로그램

**실제로 하는 일:** 다른 보안 프로그램들을 다운로드받아서 설치해주는 일종의 "설치 도우미". 그런데:
- 프로그램을 다운로드할 때 **암호화 없이(HTTP)** 받는다 → 중간에서 누군가 바꿔치기 가능
- 자동 업데이트 기능이 없다 → 취약점이 발견돼도 패치가 안 된다
- 2020년에 **북한 해커가 이걸 이용해서 악성코드를 배포**했다 (자세한 내용은 6장에서)

### AhnLab Safe Transaction (안랩) — "금융 방화벽"

**은행이 말하는 용도:** 금융 거래 중 악성 프로그램이 끼어들지 못하게 방화벽 역할

**실제로 하는 일:** 운영체제의 **가장 깊은 레벨(커널)**에서 실행되면서 네트워크 트래픽 전체를 감시한다. 즉, **당신의 PC에서 오가는 모든 인터넷 통신을 들여다볼 수 있는** 프로그램이다.

이 프로그램 자체가 뚫리면? 해커가 당신의 PC를 완전히 장악할 수 있다. 운영체제 최상위 권한으로 실행되고 있으니까.

### 이 프로그램들의 공통점

| | 일반 앱 (카카오톡, 크롬 등) | 한국 금융 보안 프로그램 |
|---|---|---|
| **권한** | 제한된 권한 (내 파일만 접근) | **운영체제 최상위 권한** (PC 전체 접근) |
| **감시 범위** | 해당 앱 내부만 | **키보드 입력 전체, 네트워크 전체, 시스템 정보 전체** |
| **설치 선택** | 안 깔아도 됨 | **안 깔면 은행 업무 불가** |
| **알 수 있는 정보** | 거의 없음 | 당신이 뭘 치는지, 어디에 접속하는지, PC에 뭐가 설치돼 있는지 |

**쉬운 비유:** 은행이 "보안을 위해 당신 집에 CCTV를 설치하겠습니다. 거실, 침실, 화장실 전부요. 아, 그리고 CCTV 시야 확보를 위해 대문도 열어놔야 합니다. 싫으면 통장을 쓸 수 없습니다"라고 하는 것과 같다.

## 5. 왜 오히려 위험한가

### 5.1 문을 많이 만들수록 도둑이 들어올 곳이 늘어난다

보안의 가장 기본적인 원칙: **문(공격 지점)을 최소화하라.**

한국 방식은 정반대다. PC에 커널 레벨 프로그램을 5~9개 설치하면, 각각이 독립적인 "문"이 된다. KAIST가 7개 프로그램을 조사했더니 **19개의 심각한 보안 구멍**을 발견했다.

비유: 집 보안을 강화한다면서 문을 9개 더 뚫어놓은 셈이다. 하나만 부실해도 도둑이 들어온다.

### 5.2 크롬이 만들어놓은 보호막을 부순다

최신 브라우저(Chrome, Safari, Firefox)는 **"샌드박스"**라는 보호 장치가 있다. 웹사이트가 당신의 PC 파일이나 프로그램에 접근하지 못하도록 격리하는 기술이다. 이게 현대 웹 보안의 핵심이다.

한국 보안 프로그램들은 이 보호막을 **일부러 뚫고** 나간다. "키보드를 감시하려면 브라우저 바깥에서 동작해야 하니까요." 그런데 보호막에 구멍을 뚫어놓으면, 해커도 그 구멍을 쓸 수 있다.

### 5.3 자물쇠를 열어서 안을 들여다본다

은행과 당신의 PC 사이의 통신은 **HTTPS(TLS)**라는 기술로 암호화되어 있다. 편지를 봉투에 넣어서 보내는 것과 같다. 중간에서 누가 편지를 가로채도 봉투를 열 수 없다.

한국 보안 프로그램들은 이 봉투를 **중간에서 열어본다**:

```
[정상적인 방식]
당신 → 🔒 암호화된 봉투 → 은행

[한국식 "보안"]
당신 → 🔒 봉투 → [보안 프로그램이 봉투를 열고 내용 확인] → 🔒 새 봉투 → 은행
```

이걸 하려면 보안 프로그램이 당신의 PC에 **가짜 자물쇠 열쇠(Root CA)**를 몰래 등록한다. 이 열쇠가 해커 손에 들어가면? **한국 전체 인터넷 사용자의 암호화 통신이 무력화**된다.

그리고 프로그램을 삭제해도 이 가짜 열쇠는 PC에 남아 있다. 지우는 기능 자체가 없다.

### 5.4 업데이트가 안 된다

KAIST가 실제 한국인 PC 48대를 조사한 결과:
- 보안 프로그램 평균 **9개** 설치
- 상당수가 **수년간 업데이트 안 된** 구버전
- 알려진 보안 구멍이 그대로 방치

Veraport는 자동 업데이트 기능 자체가 없다. 업데이트하려면 은행 웹사이트에 다시 접속해서 새 버전을 깔아야 한다. 현실적으로 아무도 하지 않는다.

비유: 경비원을 고용했는데, 경비원이 5년째 교육도 안 받고 장비도 안 바꾸고 졸고 있다. 그런데 해고할 수도 없다.

### 5.5 뭘 깔았는지 아는 사람이 없다

KAIST 설문 결과:
- 한국인 **97.4%**가 보안 프로그램을 설치 (거부하면 은행 업무 불가니까)
- **59%**는 이 프로그램이 뭘 하는지 모른다
- 선택권이 없다 — "설치 안 하면 월급 못 받아요"와 같은 구조

이건 동의가 아니다. **강제**다.

## 6. 실제로 털린 사례들

"이론적으로 위험하다"가 아니다. **실제로 일어난 일**이다.

### 사례 1: 북한 해커가 Veraport를 이용해 악성코드를 퍼뜨렸다 (2020)

**공격자:** 북한 라자루스 그룹 (국가 지원 해킹 조직)
**보고:** ESET(글로벌 보안업체), 2020년 11월

| 순서 | 무슨 일이 있었나 |
|------|-----------------|
| 1 | 은행·정부 웹사이트에서 쓰이는 Veraport가 설치된 웹사이트를 해킹 |
| 2 | 한국 보안업체 두 곳의 **디지털 인감(코드 서명 인증서)**을 훔침 |
| 3 | 북한 악성코드에 이 인감을 찍어서 "정상 프로그램"으로 위장 |
| 4 | Veraport의 자동 설치 기능을 이용해 사용자 PC에 악성코드를 배포 |
| 5 | 사용자 PC를 원격 조종할 수 있는 백도어 설치 완료 |

**핵심:** 보안을 위해 설치한 프로그램이 **악성코드 배달부** 노릇을 했다.

### 사례 2: 북한 해커가 MagicLine4NX를 뚫고 기업 네트워크에 침입했다 (2023)

**공격자:** 북한 라자루스 그룹
**경보:** **영국 정보기관(NCSC)과 한국 국가정보원(NIS)이 공동으로** 경보를 발령할 만큼 심각한 사건

| 순서 | 무슨 일이 있었나 |
|------|-----------------|
| 1 | 뉴스 웹사이트를 해킹해서 악성 코드를 심어둠 |
| 2 | 특정 기업 직원들이 그 뉴스를 읽으면, 직원 PC에 설치된 MagicLine4NX의 보안 구멍을 공격 |
| 3 | MagicLine4NX를 통해 기업 내부 네트워크에 침입 |
| 4 | 기밀 데이터 탈취, 기업 시스템 장악 |

**핵심:** 금융 인증을 위해 **의무적으로** 설치된 프로그램이 해킹 통로가 됐다. 직원에게는 이 프로그램을 안 깔 권리가 없었다.

### 사례 3: TouchEn nxKey로 전 국민 키보드를 엿볼 수 있었다 (2023)

보안 연구자 Wladimir Palant의 발견:

- 악성 웹사이트가 TouchEn nxKey를 통해 **사용자의 모든 키보드 입력을 가로챌 수 있음**
- 이 프로그램은 한국 거의 모든 PC에 깔려 있음 (Chrome 웹스토어 1,000만+ 설치)
- 즉, 하나의 취약점으로 **사실상 전 국민의 키보드를 엿볼 수 있는** 상황이었음

키로거를 막겠다고 설치한 프로그램이 **전 국민 규모의 키로거 취약점**을 만들었다.

## 7. 규제가 만든 이권 구조

### 누가 돈을 벌고 있나

| 회사 | 만든 프로그램 | 특이사항 |
|------|-------------|---------|
| **라온시큐어** | TouchEn nxKey (키보드 보안) | 2024년 매출 **533억 원**, 전년 대비 **52.6% 급증** |
| **위즈베라** | Veraport (설치 관리) | 보안 프로그램 관리 시장 사실상 독점 |
| **안랩** | Safe Transaction (방화벽) | 한국 1위 보안업체 |
| **이니텍** | INIpay (결제) | 전자결제·인증 |
| **드림시큐리티** | MagicLine4NX (금융 인증) | 북한 해커에 뚫린 그 프로그램 |

라온시큐어의 매출이 1년 만에 52.6% 급증했다. 취약점이 발견돼서 긴급 패치 계약을 따냈기 때문이다. **보안 구멍이 뚫릴수록 매출이 오르는 구조.**

### 이 시스템이 유지되는 메커니즘

```
① 금융감독원이 "보안 프로그램 설치하라"고 규정한다
       ↓
② 은행은 이 규정을 따르기 위해 특정 업체의 프로그램을 구매한다
       ↓
③ 그 업체는 규제가 보장해주는 안정적 수입을 누린다
       ↓
④ 보안 구멍이 발견되면 → 패치 계약 → 매출 증가
       ↓
⑤ 규제를 바꿀 인센티브가 아무에게도 없다
       ↓
① 으로 돌아감
```

**핵심:** 이 시스템이 유지되는 이유는 보안 때문이 아니다. **규제 포획(Regulatory Capture)** — 규제가 시민이 아니라 피규제 업체의 이익을 위해 작동하는 전형적 사례다.

- **금감원:** 규정을 바꾸면 "보안을 약화시켰다"는 비난을 받을 리스크
- **은행:** "규정대로 했으니 면책" — 바꿀 이유 없음
- **보안업체:** 규제가 매출을 보장 — 바꿀 이유 없음
- **퇴직 공무원:** 금융위원회·금감원 퇴직 후 금융결제원·인증기관 임원으로 이동하는 관행

### 사용자에게 선택권은?

없다.

- 월급 받으려면 → 은행 인터넷뱅킹 → 보안 프로그램 5개 설치
- 세금 내려면 → 국세청 홈택스 → 보안 프로그램 3개 설치
- 정부 민원 → 정부24 → 보안 프로그램 2개 설치

거부하면 서비스 자체를 사용할 수 없다. 월급을 안 받을 수도 없고, 세금을 안 낼 수도 없다.

## 8. 왜 Stripe가 안 되나

해외에서 온라인 결제를 받으려면 Stripe라는 서비스를 쓰면 된다. 코드 몇 줄이면 카드 결제가 된다. 전 세계 수백만 기업이 쓴다.

한국에서는 안 된다.

### 해외 결제 구조 (심플)

```
고객 → Stripe → Visa/Mastercard → 카드 발급 은행
(끝)
```

### 한국 결제 구조 (복잡)

```
고객 → PG사(이니시스/KCP/토스페이먼츠) → VAN사 → 개별 카드사
(각 단계마다 수수료, 각 단계마다 별도 계약)
```

한국은 Visa/Mastercard 글로벌 결제망을 **무시하고** 자체 결제망을 운영한다. 그래서:

| 장벽 | 내용 |
|------|------|
| **법인 등록** | 한국 사업자등록번호 없으면 PG사 가입 자체가 불가 |
| **허가** | 외국 기업이 한국에서 결제 서비스를 하려면 금융위원회 허가 필요 (사실상 불가) |
| **프로토콜** | 한국 카드사들이 쓰는 통신 방식이 세계 표준과 다름 |
| **본인인증** | 한국은 국제 표준(3D Secure)을 안 쓰고, 자체 인증(SMS + 공동인증서)을 씀 |

결과: 한국에서 결제를 받으려면 한국 PG사를 써야 한다. 글로벌 SaaS 기업이 한국 시장에 진출할 때 가장 큰 골칫거리 중 하나다.

## 9. 다른 나라는 어떻게 하나

### 핵심 차이: 보안을 어디에 둘 것인가

| | 한국 | 미국·유럽 |
|---|---|---|
| **기본 전제** | "사용자 PC가 위험하니까 통제하자" | "사용자 PC는 통제 불가능하니까 서버에서 막자" |
| **프로그램 설치** | 5~9개 필수 | **0개** |
| **사고 책임** | 사용자 | **금융기관** |

### 토큰화 — 카드 번호를 안 보내는 기술

해외에서 Apple Pay나 Google Pay로 결제하면, 실제 카드 번호가 가맹점에 전달되지 않는다. 대신 **일회용 가짜 번호(토큰)**를 보낸다.

해커가 이 토큰을 훔쳐도? → 한 번 쓰고 버린 번호라 아무 쓸모없다.

한국은? → PG사 서버에 카드 번호를 직접 저장하는 방식을 오래 사용했다.

### 3D Secure 2.0 — 똑똑한 인증

Visa와 Mastercard의 인증 방식:

1. **평소에 하던 패턴의 결제** → 추가 인증 없이 바로 통과 (예: 매일 쓰는 커피숍)
2. **평소와 다른 결제** (새벽 3시에 해외에서 고가 구매) → 앱 푸시 알림이나 생체인증으로 확인

프로그램 설치? 없다. 브라우저에서 다 된다.

### FIDO2/WebAuthn — 지문이나 얼굴로 인증

비밀번호 대신 **지문이나 얼굴인식**으로 로그인하는 기술. 이미 모든 주요 브라우저에 내장되어 있다.

- 피싱에 원천적으로 면역 (가짜 사이트에서는 작동 안 함)
- 별도 프로그램 설치 불필요
- Apple, Google, Microsoft 전부 지원

### 비교 — 같은 일을 할 때

| 하려는 일 | 한국에서 | 미국/유럽에서 |
|-----------|---------|-------------|
| **온라인 쇼핑 결제** | 보안 프로그램 5개 설치 → 공동인증서 → 가상 키패드 → 카드 비밀번호 앞 2자리 → SMS 인증 | 카드 번호 입력 → 끝 (필요 시 앱 푸시 한 번) |
| **은행 이체** | 보안 프로그램 5개 설치 → 공동인증서 → OTP/보안카드 | 은행 앱에서 지문 인증 → 끝 |
| **해외 사이트 결제** | 안 됨 (한국 결제 시스템이 안 붙으니까) | Stripe/카드 번호 입력 → 끝 |

## 10. 어떻게 바꿔야 하나

### 지금 당장 할 수 있는 것

**① "프로그램 깔아라" 대신 "서버에서 막아라"**

금감원 규정을 바꿔서, 은행이 글로벌 보안 표준(PCI DSS, 3D Secure 2.0)을 충족하면 클라이언트 프로그램 설치를 면제하도록 한다.

**② 지문·얼굴 인증(FIDO2)으로 전환**

공동인증서와 SMS 인증을 FIDO2로 대체. 이미 아이폰·갤럭시에 내장된 기술이다. 프로그램 설치 불필요, 피싱에도 강하다.

**③ 카드사·PG사에 PCI DSS 의무화**

국제 보안 표준 준수를 의무화하면, 서버 보안이 올라가고 클라이언트 프로그램 의존도가 낮아진다.

### 중기적으로 해야 할 것

**④ Stripe 같은 글로벌 결제사가 들어올 수 있게 시장 개방**

전자금융거래법을 바꿔서 외국 PG사가 한국에서 영업할 수 있게 한다. 경쟁이 생기면 수수료가 낮아지고 기술 혁신이 빨라진다.

**⑤ 보안 프로그램에 대한 독립적 보안 감사**

현재 금융보안원의 인증은 사실상 형식적이다. 독립적 제3자가 정기적으로 보안 감사를 하고, 취약점을 투명하게 공개하는 절차를 의무화해야 한다.

**⑥ 보안 프로그램이 암호화를 건드리지 못하게 금지**

금융 보안 프로그램이 브라우저의 HTTPS 암호화를 중간에서 풀어보는 행위를 법으로 금지해야 한다.

### 근본적으로 바꿔야 할 것

**⑦ 규제 방식 전환 — "뭘 깔아라"가 아니라 "뭘 달성해라"**

지금: "키보드 보안 프로그램을 설치해야 한다" (특정 기술을 지정)
바꾸면: "키 입력이 제3자에게 노출되지 않아야 한다" (목표만 지정)

목표만 정해주면 은행이 가장 좋은 기술을 자유롭게 선택할 수 있다. TLS 암호화, FIDO2, 서버측 보안 — 뭐든 된다. 특정 업체 프로그램을 강제할 이유가 사라진다.

**⑧ 사고 책임을 금융기관으로**

지금: "프로그램 안 깔았으니 사용자 책임"
바꾸면: "서버 보안이 부족했으니 금융기관 책임"

EU의 PSD2가 이미 이 원칙을 적용하고 있다. 책임이 금융기관에 가면, "프로그램 깔라고 했으니 우리 잘못 아냐"가 통하지 않는다. 은행이 직접 서버 보안에 투자할 수밖에 없게 된다.

**⑨ 회전문 인사 차단**

금융위·금감원 퇴직 공무원이 금융결제원·인증기관·보안업체로 이동하는 관행을 차단해야 이 이권 구조가 깨진다.

## 한 줄 요약

> 한국의 금융 보안 프로그램은 **보이스피싱을 막지 못하고**, **북한 해커에게 문을 열어줬으며**, **은행의 면책 도구로 기능하면서**, **규제가 만든 이권으로 소수 업체에 매년 수백억 원을 보장해주는** 시스템이다. 해결책은 간단하다: 다른 나라처럼 **서버에서 막고, 책임을 금융기관에 두면** 된다.
